# lyrkl

Standalone toolbox for generating phonetic lyric variations using the **APT (Adversarial PhoneTic Prompting)** method from *Bob's Confetti: Phonetic Memorization Attacks in Music and Video Generation* (Roh et al., 2025).

Lyrics are fetched from Genius, variants are generated by Claude/Gemini/OpenRouter, filtered by a 7-component phonetic similarity metric ($\Phi$), and everything is persisted in SQLite for fully incremental, idempotent operation.

---

## Quick start

```bash
# 1. Create and activate a virtual environment
python -m venv .venv
source .venv/bin/activate

# 2. Install the package
pip install -e .

# 3. Copy the env template and fill in your API keys
cp .env.example .env

# 4. Run the full pipeline on the example song list
lyrkl run-all songs.yaml

# 5. Check the database status
lyrkl status

# 6. Export for use in acelm-interp
lyrkl export --out data/prompts.json
```

## Environment variables

| Variable | Required for |
|---|---|
| `GENIUS_API_KEY` | Lyrics fetching |
| `ANTHROPIC_API_KEY` | Claude provider |
| `GEMINI_API_KEY` | Gemini provider |
| `OPENROUTER_API_KEY` | OpenRouter provider |

Get a Genius API token at [genius.com/api-clients](https://genius.com/api-clients).

## Configuration

Edit `configs/default.yaml` to change the LLM provider, model, Phi thresholds, and prompt templates. All settings have comments explaining their purpose.

```yaml
llm:
  provider: gemini       # switch from claude to gemini
  model: gemini-2.0-flash
  candidates_per_song: 3

phi:
  min_aggregate: 0.75    # stricter filtering
```

## Incremental operation

Each pipeline stage can be run independently:

```bash
lyrkl add-songs songs.yaml   # register songs without fetching
lyrkl fetch                   # pull lyrics from Genius
lyrkl style                   # generate LLM style descriptions
lyrkl vary                    # generate and filter phonetic variants
lyrkl export --out data/p.json
```

Re-running any stage is safe. The database uses content-addressed IDs and `INSERT OR IGNORE` to never duplicate work.

## Changing the prompt

Edit `prompts/apt_primary.txt`. The next `lyrkl vary` run will detect the new prompt hash and generate fresh variants, keeping the old ones in the database.

## Data layout

```
data/
  lyrkl.db                    # SQLite database
  llm_responses/
    {song_id}/
      {response_id}.txt       # raw LLM response + prompt
```

## Connection to acelm-interp

`lyrkl export` produces a JSON file in the `PromptDataset` format used by [acelm-interp](../acelm-interp). Load it directly:

```python
from acelm_interp.prompts import PromptDataset
ds = PromptDataset.load("data/prompts.json")
```
